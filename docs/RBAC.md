Контроль доступа пользователя к видео, которое ему не разрешено, в sellerproof-backend реализуется на уровне backend с использованием нескольких механизмов:

1. **Проверка принадлежности к организации**
   - При запросе видео проверяется, что `video.OrgID` совпадает с организацией пользователя.
   - Если видео принадлежит другой организации, доступ отклоняется с ошибкой "access denied".

2. **Проверка ролей и прав (RBAC)**
   - Пользователь имеет роль в организации (admin, manager, user), которая определяет его права.
   - В коде используется RBAC-система, которая проверяет разрешения пользователя по его роли.
   - Например, обычный пользователь (`roleUser`) может запрашивать только свои собственные видео (проверка поля `uploadedBy`).

3. **Проверка токенов и публичных ссылок**
   - Для публичного доступа к видео используется публичный токен, который генерируется и действует с истечением срока.
   - Если токен недействителен или срок истёк, доступ блокируется.

4. **Формирование ключей и URL**
   - Ключи объектов в Object Storage содержат `orgID` и `videoID`, что обеспечивает логическое разделение.
   - Backend контролирует, чтобы URL на загрузку/скачивание генерировались только для видео, доступ к которым разрешён пользователю.

Таким образом, гарантируется, что пользователи не смогут получить доступ к чужим данным, будь то видео других организаций или других пользователей внутри той же организации. Контроль основан на проверках в backend через организацию, роли и политики доступа.
